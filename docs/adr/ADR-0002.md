# ADR-0002: TanStack Query for Server State

**Status:** Accepted
**Date:** 2025-01-01

## Context

The frontend needs to fetch, cache, and synchronise server state (images, user data, job status). Options considered:

- **TanStack Query v5** (formerly React Query)
- **Redux RTK Query**
- **SWR**
- **Manual fetch + useState**

## Decision

We will use **TanStack Query v5** for all server state management.

Key factors:

1. **Separation of concerns:** TanStack Query handles server state; Zustand handles client/UI state — each tool does one thing well
2. **Built-in SSE support:** The `useQuery` + manual invalidation pattern works well with our Server-Sent Events job status updates
3. **Optimistic updates:** First-class support for optimistic UI — image save/delete actions update the UI immediately
4. **Cursor pagination:** `useInfiniteQuery` maps directly to our cursor-based pagination API
5. **Developer experience:** DevTools for inspecting cache state, automatic background refetch, stale-while-revalidate

## Consequences

- ✅ Automatic background refetch, cache deduplication, request deduplication
- ✅ No global state boilerplate for server data
- ✅ Optimistic updates with automatic rollback on error
- ✅ Works with React Suspense (future enhancement)
- ⚠️ Learning curve for developers unfamiliar with server-state concepts
- ⚠️ Cannot be used outside React components (use raw axios for server-side or worker code)
